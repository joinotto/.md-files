What are GitHub Shortcuts?
--------------------------

GitHub shortcuts are keyboard combinations that allow you to perform common actions quickly without relying on the mouse. This accelerates your interaction with the platform, making you more efficient, especially when managing large projects or collaborating with multiple team members.

Key GitHub Shortcuts for Effective Navigation
---------------------------------------------

Understanding how to navigate GitHub is fundamental in leveraging its full potential. Here are some shortcuts to help you move around efficiently:

*   **t:** Opens file finder
*   **g + i:** Go to issues page
*   **g + p:** Go to pull requests
*   **g + n:** Go to notifications
*   **s:** Star a repository

### Actionable Tip:

Make it a habit to use these shortcuts during your routine tasks. This minimizes reliance on the mouse and helps you navigate efficiently.

Shortcuts for Issues and Pull Requests
--------------------------------------

Managing issues and pull requests is crucial for team collaboration. Here are the essential shortcuts for navigating these features:

*   **Ctrl + Enter:** Comment on an issue/pull request
*   **Shift + s:** Subscribe/unsubscribe to notifications
*   **l:** Label an issue/pull request

### Example:

When reviewing a pull request, utilize **Ctrl + Enter** to add comments directly. This saves time spent switching between the keyboard and mouse.

Code Review and Collaboration Shortcuts
---------------------------------------

Code reviews are a vital part of team collaboration on GitHub. These shortcuts can elevate your ability to review changes effectively:

*   **.:** Open the file Editor
*   **u:** View commit history
*   **f:** Open the file finder

### Actionable Tip:

When reviewing code, utilize the **.** shortcut to edit files directly in your browser. This allows for immediate edits without navigating away from the pull request page.
